📖 README.md🇨🇳 中文版 (Chinese Version)BeamGage Pro - 智能激光光束分析系统基于 Python (PyQt5 + pyqtgraph) 构建的专业级激光光束截面分析与干涉相位重建软件。🌟 核心功能 (Core Features)硬件设备直连：基于 CTypes 直连 MVCAMSDK，支持工业相机实时高速取流。光束形貌分析：实时计算光斑能量质心、峰值强度，并支持标准的高斯拟合算法（D4σ、1/e²、刀口法）。多模态光束仿真：内置高斯、超高斯、厄米-高斯 (Hermite-Gaussian)、拉盖尔-高斯 (Laguerre-Gaussian) 涡旋光束的数学仿真模型，支持模拟指向晃动与平面波/高斯波干涉。$M^2$ 质量分析模块：内置抛物线拟合计算工具，支持手动记录多平面光斑尺寸并一键生成 SCI 级别的美观绘图。自定义图层保存：支持灵活定制导出的图像元素（十字光标、原始数据线、高斯拟合包络线、动态光阑环），并支持 CSV 原始矩阵导出。实时相位重建：内置基于傅里叶变换的离轴干涉相位提取工具，支持动态框选。🛠️ 依赖库 (Dependencies)Python 3.8+PyQt5pyqtgraphnumpyscipyMVCAMSDK (需配置对应硬件的 DLL 驱动文件)⚠️ 现阶段未解决的问题 (Known Issues & Unresolved Bugs)目前版本在复杂计算场景下仍存在部分底层架构与算法的局限性，需在后续版本中攻克：相位重建帧率瓶颈 (FPS Drops)：由于 Python 的 GIL（全局解释器锁）限制以及 Numpy 在 CPU 上执行大型矩阵 2D-FFT 的算力瓶颈。即使采用了 $2^N$ 内存对齐和主循环节流，开启相位重建后帧率仍会受到严重拖累（存在掉至 10Hz 以下的情况）。目前尚未成功实现稳定的多线程异步解耦和 GPU (CuPy) 硬件加速。相位残余倾斜与条纹伪影 (Residual Tilt & Fringes)：对于复杂的涡旋光场（包含相位奇点），当前的傅里叶寻峰与空域外差解调算法偶尔会定位偏移。这会导致相位图中残留未消干净的倾斜条纹（Tip/Tilt），或者在边缘处出现高频截断引发的吉布斯振铃（Gibbs Phenomenon）和阶数翻倍现象。3D 视图性能灾难 (3D Rendering Lag)：pyqtgraph.opengl 渲染透明遮罩的 3D 表面图消耗过大，导致严重的卡顿。为保证 2D 基础界面的可用性，目前代码中强行隐藏/移除了 3D 可视化模块。硬编码的缩放比例 (Hardcoded UI Scaling)：界面整体尺寸（1125x750）和控件字号是直接写死的，在不同分辨率或 DPI 缩放比例的显示器上可能会出现控件挤压或错位。强绑定的算法逻辑：当前处理循环 render_process_loop 中，图像阈值截断（Clipping）与 FFT 相位解算的耦合度过高，容易导致正弦干涉条纹的波形被破坏（削顶效应），从而诱发二次谐波干扰。🇬🇧 英文版 (English Version)BeamGage Pro - Laser Beam Profiler & Phase AnalyzerA professional-grade laser beam profiling and interferometric phase reconstruction application built on Python (PyQt5 + pyqtgraph).🌟 Core FeaturesDirect Hardware Access: Real-time high-speed streaming via CTypes binding to MVCAMSDK.Beam Profiling: Real-time calculation of beam centroid, peak intensity, and beam widths using standard fitting algorithms (D4σ, 1/e², Knife-edge).Multi-modal Simulation: Built-in mathematical simulators for Gaussian, Super-Gaussian, Hermite-Gaussian, and Laguerre-Gaussian (vortex) beams, including beam jitter and interference emulation.$M^2$ Beam Quality Tool: Dedicated module to record multi-plane beam waists, perform parabolic fitting, and export SCI-publication-ready plots.Customizable Overlay Export: Flexible saving options allowing users to selectively render crosshairs, raw profiles, fitted Gaussian curves, and ROI rings onto exported images, alongside raw CSV data export.Real-time Phase Retrieval: Fourier-transform-based off-axis interferometric phase extraction for dynamically selected Regions of Interest (ROI).🛠️ DependenciesPython 3.8+PyQt5pyqtgraphnumpyscipyMVCAMSDK (Hardware-specific DLL required)⚠️ Known Issues & Unresolved BugsThe current build has several architectural and algorithmic limitations under heavy computational loads:Phase Reconstruction FPS Bottleneck: Due to Python's GIL and CPU limitations when performing large matrix 2D-FFT via Numpy, enabling phase reconstruction severely degrades the application's framerate (sometimes dropping below 10Hz). Asynchronous multi-threading and GPU hardware acceleration (CuPy) have not been successfully or stably integrated.Residual Tilt and Fringe Artifacts: In complex vortex fields containing phase singularities, the current Fourier peak-finding and spatial heterodyne demodulation algorithms occasionally miscalculate the carrier frequency. This results in residual tilt fringes, phase wrapping errors, or Gibbs phenomenon (fringe splitting/doubling) caused by hard edge truncation.3D Visualization Lag: The pyqtgraph.opengl 3D surface rendering with transparent masks creates catastrophic performance drops. To maintain the usability of the 2D interface, the 3D visualization components are currently hidden/disabled.Hardcoded UI Scaling: The window dimensions (1125x750) and font sizes are hardcoded. The application may experience layout compression or misalignment on monitors with different resolutions or DPI scalings.Coupled Algorithm Logic: In the main render_process_loop, the image threshold clipping is tightly coupled with the raw data fed into the FFT phase solver. This can inadvertently clip the bottom of sine-wave interference fringes, generating strong second-harmonic frequencies that confuse the demodulation algorithm.
